\section{Интерактивная визуализация и обработка всех углов}

В предыдущих разделах мы научились загружать, усреднять и фильтровать терагерцовые сигналы, а также вычислять их спектры и пропускания для отдельного выбранного угла. Но в реальном физическом эксперименте мы всегда имеем дело с целым семейством углов (например, от $0^\circ$ до $90^\circ$ с определенным шагом). Анализировать каждый угол по отдельности, постоянно переписывая пути к файлам вручную — крайне неэффективный подход, ведущий к ошибкам.

В этом разделе мы решим эту проблему фундаментально и профессионально:
\begin{enumerate}
    \item Разработаем архитектуру локальной базы данных прямо в оперативной памяти (СУБД на основе хэш-таблиц), в которой все файлы сигналов и фонов будут строго связаны по метаданным.
    \item Реализуем модуль автоматического пакетного (массового) импорта и обработки всех измерений с учетом физического дрейфа прибора (медленного изменения характеристик лазера и влажности во времени).
    \item Создадим быстродействующий интерактивный графический интерфейс с сеткой графиков (plot grid), объединяющий отображение временного импульса и его спектра с мгновенным переключением углов на лету без мерцания экрана.
\end{enumerate}

\subsection{Архитектура СУБД для терагерцовых измерений}

В типичных студенческих скриптах обработка данных часто превращается в хаос: создаются десятки независимых массивов вида \texttt{signals\_raw}, \texttt{backgrounds\_raw}, \texttt{transmissions}. При этом связь между файлом сигнала образца и соответствующим ему референсным фоном (background) теряется или жестко зашивается через индексы, что приводит к критическим ошибкам при добавлении новых измерений.

Чтобы решить эту проблему раз и навсегда, мы применим профессиональный архитектурный паттерн — \textbf{систему управления базами данных (СУБД)}. В нашем случае это будет простая и быстрая реляционная (связанная) база данных, живущая прямо в оперативной памяти компьютера в виде единого централизованного словаря \texttt{data\_store}.

\subsubsection{Схема составных ключей}

Ключом в нашем словаре-хранилище \texttt{data\_store} будет строго типизированный кортеж (tuple — неизменяемый упорядоченный список) из четырех элементов, однозначно идентифицирующий любое измерение:
\begin{equation}
\text{Key} = (\text{Tag}, \text{Angle}_1, \text{Angle}_2, \text{Repetition})
\end{equation}

Где каждый элемент имеет четкий физический и логический смысл:
\begin{enumerate}
    \item \textbf{\texttt{Tag} (строка):} Уникальный идентификатор типа данных:
    \begin{itemize}
        \item \texttt{'signal\_raw'} — сырой временной ТГц-сигнал образца для конкретного прохода.
        \item \texttt{'bg\_raw'} — сырой временной референсный сигнал (фон) для конкретного прохода.
        \item \texttt{'spec\_avg'} — итоговый усредненный спектр пропускания образца по мощности (без реплик).
        \item \texttt{'spec\_bg\_avg'} — итоговый усредненный спектр мощности опорного сигнала (фона).
    \end{itemize}
    \item \textbf{\texttt{Angle1} (число с плавающей точкой):} Нормализованный угол установки первого ротатора (поляризатора) в градусах.
    \item \textbf{\texttt{Angle2} (число с плавающей точкой):} Угол установки второго ротатора (анализатора) в градусах.
    \item \textbf{\texttt{Repetition} (целое число или \texttt{None}):} Номер повторного измерения (прохода / реплики). Для усредненных данных, где реплики уже объединены, этот параметр равен \texttt{None}.
\end{enumerate}

\subsubsection{Декларативная SQL-like фильтрация}

Одно из главных преимуществ такого СУБД-подхода — невероятно простая и элегантная фильтрация данных с помощью встроенных в Python генераторов словарей (dict comprehensions — компактного синтаксиса сборки новых коллекций на лету). Это заменяет нам сложные SQL-запросы.

Например, извлечь из базы данных все усредненные спектры образцов для построения графиков можно всего одной лаконичной строчкой:
\begin{minted}{python}
# Фильтрация словаря по первому элементу кортежа-ключа:
spectra_to_plot = {
    k: v 
    for k, v in data_store.items() 
    if k[0] == 'spec_avg'
}
\end{minted}

Конструкция \texttt{data\_store.items()} возвращает пары «ключ-значение». Наш фильтр проверяет первый элемент ключа \texttt{k[0]} (строку-тег) и оставляет только те записи, у которых тег равен \texttt{'spec\_avg'}.

\subsection{Разработка загрузчика базы данных: Учет дрейфа прибора}

Условия проведения физического эксперимента непрерывно меняются во времени: колеблется температура и комнатная влажность воздуха, дрейфует (плавно изменяется) средняя мощность накачки фемтосекундного лазера. По этой причине каждый замер сигнала образца жестко привязан к конкретному файлу фона, измеренному непосредственно в той же серии проходов.

\warning{\textbf{Критическая физическая ошибка:} Абсолютно недопустимо сначала усреднить отдельно все сигналы образца и отдельно все фоны во временной области, а затем делить их спектры друг на друга. Единственный физически грамотный путь:\\1. Рассчитать спектры сигнала и парного ему фона строго в рамках одного прохода (реплики).\\2. Разделить их мощности друг на друга, получив индивидуальное пропускание конкретного прохода: $T_i(\nu) = |E_{\text{sig}, i}(\nu)|^2 / |E_{\text{bg}, i}(\nu)|^2$. Деление мгновенно скомпенсирует дрейф мощности лазера!\\3. И только затем усреднить спектры пропускания по всем повторениям: $T_{\text{avg}}(\nu) = \frac{1}{N} \sum T_i(\nu)$.\\Наша СУБД-структура обязана строго поддерживать этот алгоритм.}

Для реализации этого механизма мы обновим модуль \texttt{data\_loader.py}, добавив функцию \texttt{load\_dataset\_to\_store}. Она автоматически обходит папку с данными, находит пары файлов по метаданным в именах и сохраняет их в базу данных. Полный код этой функции приведен в \textbf{\textit{Приложение~\ref{app:data_loader}}}.

Давай подробно разберем логику работы этого загрузчика:
\begin{minipage}{\linewidth}
	\captionof{listing}{Реляционная загрузка данных в базу (data\_loader.py)}
	\begin{minted}{python}
def load_dataset_to_store(data_dir) -> dict:
    data_store = {}
    path = Path(data_dir)
    
    # Шаг 1. Предварительно загружаем все фоновые файлы по их именам
    bg_files = {}
    for filepath in path.glob("*.txt"):
        if filepath.name.startswith("bg"):
            bg_id = filepath.stem  # например, 'bg_1'
            t, E = load_tds_data(filepath)
            bg_files[bg_id] = (t, E)
            
    # Шаг 2. Сканируем файлы сигналов образцов и привязываем их к фонам
    for filepath in path.glob("*.txt"):
        filename = filepath.name
        if filename.startswith("bg"):
            continue
            
        try:
            angle1, angle2, rep, bg_name = parse_filename(filename)
        except ValueError:
            continue
            
        t, E_signal = load_tds_data(filepath)
        
        # Сохраняем сырой сигнал образца в СУБД под составным ключом
        data_store[('signal_raw', angle1, angle2, rep)] = (t, E_signal)
        
        # Связываем этот проход с конкретным сырым фоном
        if bg_name in bg_files:
            t_bg, E_bg = bg_files[bg_name]
            data_store[('bg_raw', angle1, angle2, rep)] = (t_bg, E_bg)
            
    return data_store
	\end{minted}
\end{minipage}

\note{Программа сканирует папку в два прохода. На первом шаге она загружает все референсные (фоновые) сигналы в словарь \texttt{bg\_files}. На втором шаге она разбирает имена файлов сигналов образца, извлекает углы и номер повторения \texttt{rep}, а затем формирует в \texttt{data\_store} неразрывные пары сигналов и фонов с идентичными метаданными в ключах.}

\subsection{Пакетная обработка спектров (Batch Processing)}

Теперь, когда все сырые данные разложены по полочкам в нашей СУБД, напишем функцию для их автоматической пакетной обработки. Функция должна пройтись по всем уникальным углам, рассчитать спектры пропускания для каждого повторения, а затем корректно усреднить их по частоте.

Добавим функцию \texttt{process\_dataset} в наш спектральный модуль \texttt{spectrum.py} (полный листинг модуля находится в \textbf{\textit{Приложение~\ref{app:spectrum}}}):

\begin{minipage}{\linewidth}
	\captionof{listing}{Пакетная обработка базы данных (spectrum.py)}
	\begin{minted}{python}
def process_dataset(data_store: dict) -> dict:
    # Находим все уникальные пары углов (angle1, angle2)
    angle_pairs = set()
    for key in data_store.keys():
        if key[0] == 'signal_raw':
            angle_pairs.add((key[1], key[2]))
            
    # Для каждой пары углов проводим вычисления
    for angle1, angle2 in sorted(angle_pairs):
        transmissions = []
        bg_spectra = []
        freqs_common = None
        
        reps = [key[3] for key in data_store.keys() 
                if key[0] == 'signal_raw' and key[1] == angle1 and key[2] == angle2]
                
        for rep in sorted(reps):
            sig_key = ('signal_raw', angle1, angle2, rep)
            bg_key = ('bg_raw', angle1, angle2, rep)
            
            if bg_key in data_store:
                t_sig, E_sig = data_store[sig_key]
                t_bg, E_bg = data_store[bg_key]
                
                # Рассчитываем пропускание конкретной реплики
                freqs, spec_sig, spec_bg, transmission = calculate_transmission(
                    t_sig, E_sig, t_bg, E_bg
                )
                
                transmissions.append(transmission)
                bg_spectra.append(spec_bg)
                if freqs_common is None:
                    freqs_common = freqs
                    
        if transmissions:
            # Математическое усреднение по повторениям для компенсации дрейфа
            trans_avg = np.mean(transmissions, axis=0)
            bg_avg = np.mean(bg_spectra, axis=0)
            
            # Сохраняем агрегированные данные обратно в СУБД
            data_store[('spec_avg', angle1, angle2, None)] = (freqs_common, trans_avg)
            data_store[('spec_bg_avg', angle1, angle2, None)] = (freqs_common, bg_avg)
            
    return data_store
	\end{minted}
\end{minipage}

\note{Благодаря NumPy векторным операциям усреднение семейства спектров выполняется мгновенно с помощью функции \texttt{np.mean(..., axis=0)}. Это позволяет обрабатывать огромные экспериментальные массивы за доли секунды.}

\subsection{Интерактивный переключатель углов с сеткой графиков (Plot Grid)}

Перейдем к созданию графического интерфейса. Мы разработаем полноценную интерактивную панель, использующую концепцию \textbf{сетки графиков} (plot grid). Окно программы будет разделено на две части:
\begin{enumerate}
    \item \textbf{Верхняя панель} — временной ТГц-сигнал образца (сырой и заокненный) с наложенной огибающей Гауссова окна.
    \item \textbf{Нижняя панель} — соответствующий ему спектр пропускания в рабочем частотном диапазоне.
\end{enumerate}

При клике на радиокнопку выбора угла оба графика должны обновляться одновременно и без мерцания.

\subsubsection{Профессиональное обновление данных в Matplotlib}

\warning{Никогда не вызывайте \texttt{ax.clear()} внутри функций обратного вызова (callbacks) интерактивных виджетов. Очистка осей сбрасывает все подписи, сетку, масштабы и приводит к сильному мерцанию интерфейса. Профессиональный подход — построить линии один раз с помощью метода \texttt{ax.plot()} при инициализации, сохранить ссылки на объекты \texttt{Line2D}, а затем быстро обновлять их координаты по оси Y с помощью метода \texttt{line.set\_ydata()}.}

Давай создадим в корне нашего проекта файл \texttt{interactive\_plots.py} и реализуем в нем это интерактивное микро-приложение. Полный код модуля доступен в \textbf{\textit{Приложение~\ref{app:interactive_plots}}}.

Рассмотрим ключевые моменты инициализации сетки графиков и функции обновления данных:

\begin{minipage}{\linewidth}
	\captionof{listing}{Логика обновления сетки графиков (interactive\_plots.py)}
	\begin{minted}{python}
# Создаем сетку из двух графиков (2 строки, 1 колонка)
fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(11, 8))
plt.subplots_adjust(left=0.25, hspace=0.35) # освобождаем место слева

# Строим начальные графики и сохраняем ссылки на объекты Line2D
line_raw, = ax1.plot(t_sig, E_sig_dc, color='gray', alpha=0.5, label='Исходный')
line_win, = ax1.plot(t_sig, E_sig_win, color='royalblue', label='Заокненный')
line_env_pos, = ax1.plot(t_sig, win * max_amp, 'r--', label='Окно')
line_env_neg, = ax1.plot(t_sig, -win * max_amp, 'r--')

mask = (freqs >= config.F_MIN) & (freqs <= config.F_MAX)
line_trans, = ax2.plot(freqs[mask], transmission[mask], color='darkgreen')

# Функция обновления данных (вызывается при клике на виджет)
def update_plot(label):
    key = label_to_key[label]
    a1, a2 = key[1], key[2]
    
    # 1. Обновляем временную область
    t_new, E_new = data_store[('signal_raw', a1, a2, 1)]
    E_new_dc = spectrum.remove_dc(E_new)
    E_new_win, win_new = spectrum.apply_gaussian_window(t_new, E_new_dc)
    max_amp_new = np.max(np.abs(E_new_dc))
    
    line_raw.set_ydata(E_new_dc)
    line_win.set_ydata(E_new_win)
    line_env_pos.set_ydata(win_new * max_amp_new)
    line_env_neg.set_ydata(-win_new * max_amp_new)
    
    ax1.set_title(f"Временная область. Угол 1: {a1}°")
    ax1.relim()
    ax1.autoscale_view(scalex=False, scaley=True)
    
    # 2. Обновляем частотную область (спектр пропускания)
    freqs_new, trans_new = data_store[key]
    mask_new = (freqs_new >= config.F_MIN) & (freqs_new <= config.F_MAX)
    
    line_trans.set_xdata(freqs_new[mask_new])
    line_trans.set_ydata(trans_new[mask_new])
    
    ax2.set_title(f"Частотная область. Угол 1: {a1}°")
    ax2.relim()
    ax2.autoscale_view(scalex=False, scaley=True)
    
    # Запрашиваем перерисовку холста (холст обновится плавно)
    fig.canvas.draw_idle()
	\end{minted}
\end{minipage}

\note{Методы \texttt{ax.relim()} и \texttt{ax.autoscale\_view()} заставляют Matplotlib автоматически пересчитать новые границы графиков под изменившийся сигнал, но делают это выборочно — мы отключили автомасштабирование по оси X (\texttt{scalex=False}), чтобы временная шкала и диапазон частот оставались стабильными, а масштабировалась только амплитуда.}

\subsection{Визуализация семейства спектров}

Помимо поштучной инспекции углов, физику важно увидеть общую картину — семейство всех полученных спектров на одной координатной оси. Это позволит проследить плавную угловую модуляцию ТГц излучения.

Напишем вспомогательную функцию \texttt{plot\_all\_transmissions()}, которая извлекает из СУБД все усредненные спектры и строит их, используя плавную цветовую карту градиента:

\begin{minipage}{\linewidth}
	\captionof{listing}{Построение семейства кривых пропускания (interactive\_plots.py)}
	\begin{minted}{python}
def plot_all_transmissions():
    data_store = data_loader.load_dataset_to_store(config.DATA_DIR)
    data_store = spectrum.process_dataset(data_store)
    
    spectra_keys = sorted([k for k in data_store.keys() if k[0] == 'spec_avg'])
    
    plt.figure(figsize=(10, 6))
    
    # Создаем красивый градиент цветов от фиолетового к оранжевому
    colors = plt.cm.plasma(np.linspace(0, 0.85, len(spectra_keys)))
    
    for key, color in zip(spectra_keys, colors):
        freqs, transmission = data_store[key]
        mask = (freqs >= config.F_MIN) & (freqs <= config.F_MAX)
        plt.plot(freqs[mask], transmission[mask], label=f"Угол {key[1]}°", color=color, linewidth=1.5)
        
    plt.xlim(config.F_MIN, config.F_MAX)
    plt.ylim(-0.05, 1.1)
    plt.xlabel("Частота (ТГц)")
    plt.ylabel("Коэффициент пропускания по мощности")
    plt.title("Семейство спектров пропускания для различных углов")
    plt.grid(True, linestyle='--', alpha=0.5)
    plt.legend(loc='lower left', ncol=2)
    plt.tight_layout()
    plt.show()
	\end{minted}
\end{minipage}

Использование конструкции \texttt{plt.cm.plasma(np.linspace(...))} позволяет автоматически распределить цвета линий по спектральной карте. Это делает графики эстетичными и легко читаемыми. Итоговый график семейства кривых пропускания показан на \figref{fig:5_2}.

\subsection{Итоги раздела}

В этом разделе мы совершили качественный инженерный скачок в обработке данных:
\begin{enumerate}
    \item Построили быструю СУБД в оперативной памяти на базе словарей с составными кортеж-ключами \texttt{(Tag, Angle1, Angle2, Repetition)}, исключив путаницу в файлах.
