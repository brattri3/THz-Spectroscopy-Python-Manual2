\section{Интегрированное сквозное приложение обработки данных} \label{sec:integrated_app}

В предыдущих разделах мы подробно изучили отдельные этапы обработки экспериментальных данных терагерцовой спектроскопии: от загрузки сырых временных сигналов до численной оптимизации геометрических параметров проволочных решёток на одной частоте и анализа их спектральной зависимости. Однако на практике исследователю крайне неудобно запускать несколько разрозненных скриптов, переносить параметры вручную через конфигурационные файлы и терять визуальную связь между этапами.

В данном разделе мы объединим все разработанные алгоритмы в единое, сквозное многоэтапное интерактивное приложение \mycode{main.py}. Это микро-приложение последовательно ведёт пользователя по четырём ключевым стадиям анализа, интерактивно передавая результаты и настройки с каждого предыдущего шага на последующий.

\subsection{Концепция последовательной передачи параметров}

Одной из главных проблем при разработке многооконных приложений на чистом Python без использования тяжеловесных фреймворков (таких как PyQt или Tkinter) является организация переходов между экранами и передача контекста. В библиотеке \mycode{matplotlib} эта задача решается элегантно и просто с помощью концепции \textbf{последовательного вызова окон (Pipeline Pattern)}.

Мы разбиваем процесс обработки на четыре изолированные функции-шага, каждая из которых:
\begin{enumerate}
    \item Принимает на вход данные и параметры с предыдущего шага.
    \item Создаёт интерактивное окно \mycode{matplotlib} со слайдерами, кнопками и радиокнопками.
    \item Позволяет исследователю визуально настроить параметры (ширину фильтра, шаг интерполяции, начальные приближения).
    \item При нажатии кнопки «Далее» (которая программно вызывает закрытие окна через \mycode{plt.close(fig)}) сохраняет текущие значения виджетов и возвращает их в виде словаря параметров в вызывающий код.
\end{enumerate}

Таким образом, главная функция \mycode{main()} превращается в простой и прозрачный конвейер:
\begin{minted}{python}
def main():
    signals_raw, backgrounds = load_first_repetition_dataset(config.DATA_DIR)
    
    # Шаг 1: Фильтрация во временной области на первом повторении сигналов
    win_params = run_step1_time_domain(signals_raw, backgrounds)
    
    # Загружаем полную сырую БД для физически корректного усреднения спектров в частотной области
    data_store = data_loader.load_dataset_to_store(config.DATA_DIR)
    
    # Шаг 2: Спектральный анализ на усредненных по всем проходам спектрах
    dsp_params = run_step2_frequency_domain(data_store, win_params)
    
    # Расчет спектров по настроенным на предыдущих шагах параметрам
    spectra = compute_spectra_with_params(
        data_store, 
        pad_factor=dsp_params['pad_factor'], 
        sigma=dsp_params['sigma']
    )
    
    # Шаг 3: Сопоставление с теорией на фиксированной частоте
    fit_params = run_step3_interactive_fit(spectra, dsp_params['pad_factor'])
    
    # Шаг 4: Разреженная пакетная спектральная оптимизация параметров
    run_step4_batch_optimization(spectra, fit_params)
\end{minted}

\subsubsection{Разбор кода: Главный конвейер main()}

Разберем структуру и последовательность вызовов в главной функции-оркестраторе:

\begin{syntaxdesc}
    \item[Загрузка сырых сигналов \mycode{load_first_repetition_dataset}] Функция загрузки первой реплики сигналов образца и фона для калибровки временного окна на шаге 1.
    \item[Загрузка полной базы данных \mycode{load_dataset_to_store}] Загружает полную сырую БД в память для последующего попарного расчета спектров и их усреднения на частотной сетке.
    \item[Временная фильтрация \mycode{run_step1_time_domain}] Вызывает интерактивное окно шага 1 для выбора оптимальной полуширины временного фильтра Гаусса.
    \item[Спектральный расчет \mycode{run_step2_frequency_domain}] Запускает окно шага 2 для оценки влияния коэффициента дополнения нулями (Pad Factor).
    \item[Сопоставление на частоте \mycode{run_step3_interactive_fit}] Вызывает интерактивный стенд шага 3 для сопоставления модели Бланко с экспериментом.
    \item[Пакетная оптимизация \mycode{run_step4_batch_optimization}] Выполняет финальный цикл подгонки геометрических параметров по всему полезному спектру.
\end{syntaxdesc}

\subsection{Описание этапов сквозной обработки}

Рассмотрим подробно каждый из четырёх интерактивных шагов, реализованных в нашем сквозном приложении.

\subsubsection{Шаг 1: Настройка Гауссова окна во временной области}

Первое окно приложения предназначено для первичного анализа сигналов во временной области и настройки временного окна пропускания на основе импульсов первого повторного измерения (Рисунок~\ref{fig:9_1}). В отличие от последующих этапов, здесь отображаются исходные, неусредненные импульсы для каждого угла (первый проход, соответствующий цифре «1» в имени файла).

Студент выбирает исследуемый угол с помощью радиокнопок \mycode{RadioButtons}. На верхнем графике одновременно отображаются исходный временной сигнал и сигнал, умноженный на Гауссово окно. В отличие от упрощенных реализаций, сдвиг центра окна больше не настраивается вручную — он вычисляется автоматически по положению максимума амплитуды референтного фона для достижения наилучшей стабильности. Единственным регулируемым параметром является:
\begin{itemize}
    \item \textbf{Ширина окна $\sigma$ (пс)} — слайдер настраивает длительность пропускаемого интервала в диапазоне от 5 до 500 пс. Слишком узкое окно обрезает спектр и искажает сигнал, а слишком широкое — захватывает паразитные эхо-отражения.
\end{itemize}

Для того чтобы студент мог мгновенно оценить спектральные последствия временного окна, на нижнем графике в реальном времени отображается БПФ-спектр исходного и отфильтрованного сигналов. При нажатии кнопки «Далее» настроенная ширина $\sigma$ передается на следующий шаг.

\subsubsection{Шаг 2: Спектральный анализ и выбор коэффициента дополнения нулями}

На втором шаге данные усредняются по всем повторным измерениям для каждого угла, и происходит вычисление спектров пропускания (Рисунок~\ref{fig:9_2}).

Вместо отображения одного выбранного угла, на втором шаге строится полное семейство кривых пропускания для всех измеренных углов в двух шкалах одновременно:
\begin{itemize}
    \item \textbf{Слева} — спектр пропускания по мощности в процентах ($100 \times T(\nu)$).
    \item \textbf{Справа} — спектр пропускания по мощности в децибелах ($10 \log_{10} T(\nu)$).
\end{itemize}

Главным регулируемым параметром выступает \textbf{Pad Factor} (слайдер от 1 до 8), изменяющий степень дополнения нулями. Студент наглядно видит, как увеличение плотности спектральных точек сглаживает графики пропускания для всего ансамбля кривых одновременно. Выбранный \mycode{pad_factor} сохраняется и передается в дальнейшие расчеты.

\smartfigure{fig9.2.png}{Визуализация спектров амплитуды и результирующего пропускания с регулировкой дополнения нулями (Pad Factor).}{fig:9_2}

\subsubsection{Шаг 3: Сопоставление теории и эксперимента на фиксированной частоте}

Третий шаг — это ядро лабораторной работы, где осуществляется сопоставление экспериментальных результатов с теоретической моделью Бланко для двух проволочных поляризаторов (Рисунок~\ref{fig:9_3}). Экспериментальные углы установки ротатора предварительно преобразуются к диапазону $[-180, 180]$ градусов, что позволяет получить угловую зависимость от $-90^\circ$ до $90^\circ$.

В связи с этим в модель вводится шестой интерактивный параметр подгонки — \textbf{систематический сдвиг угла установки поляризатора в ротаторе} ($\theta_{\text{offset}}$), настраиваемый с помощью нового слайдера в диапазоне от $-15^\circ$ до $15^\circ$. Теоретическая кривая рассчитывается по формуле Бланко с аргументом $\theta - \theta_{\text{offset}}$.

Таким образом, окно содержит шесть слайдеров: частоту анализа $\nu$, шаг решётки $P$, диаметр проволоки $D$, масштабный множитель периода $P_{\text{scale}}$, коэффициент затухания $Loss$ и сдвиг угла $\theta_{\text{offset}}$. Кнопка «Подогнать $\nu$» запускает пятипараметрическую оптимизацию Нелдера-Мида на выбранной частоте, автоматически подгоняя все параметры (включая угловой сдвиг) под экспериментальные точки.

\smartfigure{fig9.3.png}{Сравнение теории Бланко с экспериментальными данными пропускания от угла на фиксированной частоте с функцией локальной оптимизации.}{fig:9_3}

\subsubsection{Шаг 4: Пакетная спектральная оптимизация}

После нажатия кнопки «Далее» на третьем этапе приложение автоматически переходит к финальной стадии — пакетной оптимизации параметров по всему спектру частот в диапазоне $0.2 - 1.8$ ТГц (Рисунок~\ref{fig:9_4}).

Поскольку спектр содержит около 1600 точек, проведение полной Nelder-Mead оптимизации по 5 параметрам для каждой точки заняло бы слишком много времени. Для ускорения и повышения стабильности расчетов в приложении реализован \textbf{разреженный шаг оптимизации}: расчет производится с шагом в 20 спектральных точек (или через каждые 10--100 точек).

Оптимизатор последовательно рассчитывает наиболее вероятные значения пяти параметров: $P$, $D$, $P_{\text{scale}}$, $Loss$ и $\theta_{\text{offset}}$. В качестве начального приближения для каждой частоты используются оптимальные параметры, найденные на предыдущем шаге.

Результатом работы является сетка из 6 графиков (3 ряда, 2 колонки): частотные зависимости шага решетки $P$, диаметра проволоки $D$, масштабного коэффициента $P_{\text{scale}}$, потерь $Loss$, систематического смещения угла $\theta_{\text{offset}}$ и RMSE подгонки. Над графиками выводятся итоговые спектрально-усредненные физические параметры решетки.

\smartfigure{fig9.4.png}{Частотная зависимость оптимальных геометрических параметров проволочной решетки и коэффициента потерь.}{fig:9_4}

\subsection{Итоги раздела}

Созданное нами интегрированное сквозное приложение представляет собой полноценный программный комплекс для научных исследований в области терагерцовой оптики. Благодаря использованию объектно-ориентированных виджетов Matplotlib и концепции последовательного конвейера, мы получили мощный, но в то же время простой в понимании и модификации инструмент, не требующий сторонних интерфейсных библиотек. Студент получает уникальную возможность проследить весь путь обработки сигнала: от «сырой» осциллограммы на экране осциллографа до прецизионного определения размеров микроструктур, лежащих далеко за пределом оптического разрешения обычных микроскопов.
